Introdução

Este projeto pesquisa como HTML (front-end), bancos de dados e aplicativos mobile trabalham juntos em sistemas reais. O site apresenta conceitos, fluxo de integração e exemplos práticos (ex.: iFood, Nubank).

O que é HTML e como funciona no front-end

HTML (HyperText Markup Language) é a linguagem de marcação usada para estruturar conteúdo na web. No front-end, o HTML define a estrutura (títulos, parágrafos, listas, formulários). O HTML é combinado com CSS (estilo) e JavaScript (comportamento) para criar páginas interativas.

  • HTML: estrutura (semântica)
  • CSS: aparência e layout responsivo
  • JavaScript: interações, chamadas a APIs

O que é um banco de dados e por que é importante

Um banco de dados armazena e organiza dados de forma persistente. Ele permite buscar, inserir, atualizar e excluir informações usadas pelo sistema: dados de usuários, pedidos, transações, inventário, etc.

Tipos comuns:

  • Relacionais (SQL): MySQL, PostgreSQL — tabelas com relações.
  • NoSQL: MongoDB, Redis — documentos, chave-valor, ideal para alta escalabilidade e dados não estruturados.

Por que é importante: integridade dos dados, consultas eficientes e persistência entre sessões e dispositivos.

O que são aplicativos mobile e como se comunicam com servidores

Aplicativos mobile (Android, iOS) são programas instalados em celulares. Para obter/guardar dados, eles se comunicam com servidores via rede (HTTP/HTTPS). Essa comunicação normalmente usa APIs REST ou GraphQL que expõem endpoints (URLs) que o app consome.

Fluxo de comunicação básico:

  1. App envia requisição (ex.: POST /login) via HTTPS.
  2. Servidor processa, consulta o banco de dados, gera resposta (JSON).
  3. App recebe JSON e atualiza a interface para o usuário.

Como HTML, banco de dados e mobile se integram em sistemas reais

Resumo do fluxo completo:

  1. Front-end (site em HTML/CSS/JS) e app mobile enviam requisições HTTP para o backend (API).
  2. O backend (servidor) autentica, processa lógica de negócio e usa o banco de dados para ler/escrever dados.
  3. O banco de dados persiste as informações (usuário, pedido, histórico).
  4. O backend responde com dados (JSON); front-end ou app renderizam para o usuário.

Componentes típicos:

  • Cliente web (HTML/CSS/JS) — visualização para navegadores
  • App mobile — experiência nativa
  • API/Servidor — lógica e segurança
  • Banco de dados — persistência

Exemplos práticos

iFood (exemplo de app de delivery)

Fluxo simplificado:

  1. O usuário abre o app (mobile) ou site (HTML) e busca restaurantes.
  2. O app faz requisição à API para listar restaurantes e menus.
  3. Ao criar um pedido, o backend grava o pedido no banco (pedidos, itens, endereço).
  4. O sistema notifica restaurante e entregador (websockets/ push notifications).
  5. Pagamentos podem ser tratados por gateways externos (ex.: PagSeguro, Stripe) e o resultado registrado no banco.

Nubank (exemplo de app bancário)

Fluxo simplificado:

  1. App mobile requisita saldo/últimas transações via API segura (HTTPS + autenticação forte).
  2. Servidor autentica o usuário (tokens JWT, sessões), consulta banco de dados relacional e retorna dados criptografados.
  3. Operações financeiras passam por validações, controle de concorrência e armazenam logs e transações no BD.
  4. Notificações em tempo real podem usar push notifications ou streams.
